# 設計決策紀錄（ADR）

> 記錄關鍵架構決策、理由、以及被否決的替代方案。每條對應研究證據。

---

## ADR-001：定位為「羅盤」而非「預言機」
- **決策**：產品不預測個體命運，只做「群體概率 + 結構化反思」。
- **理由**：拉普拉斯妖認識論上不可能（RF §1）；個體高後果預測天花板 R²<0.2（RF §2）。
- **否決的替代**：「AI 算命/AI 八字預測命運」——無證據支持，且觸法巴納姆陷阱。

## ADR-002：兩層之間設硬性防火牆
- **決策**：Layer 1 概率與 Layer 2 符號反思物理隔離，輸出永不互灌。
- **理由**：符號神祕感會讓人高估概率；概率數字會讓符號像預言。混合 = 互相污染。
- **狀態**：架構強制，UI 必須視覺區隔。

## ADR-003：易經僅作「易讀符號介面」，零計算/零預測角色
- **決策**：64 卦只當觸發反思的查表式符號標籤。
- **理由**：易經數學內核真實但**無計算價值**——王文卦序訓練神經網路反而變差（RF §3）；
  「對應遺傳密碼」等延伸主張是數字命理附會（已駁回）。
- **否決的替代**：用卦象結構做特徵工程/演算法——研究實測無效。

## ADR-004：Layer 2 引擎是「去偏誤協議」，不是占卜
- **決策**：核心是 consider-the-opposite + premortem + outside view + 決策日誌。
- **理由**：這是去偏誤文獻**唯一有 RCT 證據**的東西（Morewedge 2015，RF §4）。
- **限制接受**：去偏誤遷移弱、真實生活效果未證實（RF §4）→ 故用持續性決策日誌補救。

## ADR-005：去巴納姆防護為不可妥協需求
- **決策**：禁通用模糊語句、強制證偽、明示無預測力、可證偽決策日誌。
- **理由**：Forer 效應證明通用描述會製造假「準確感」（RF §3）。這是不淪為占卜騙局的命門。

## ADR-006：先做驗證實驗，再寫產品
- **決策**：核心假設用 RCT 驗證後才開發（見 MVP-EXPERIMENT.md）。
- **理由**：「符號反思能否在不誘發巴納姆下改善決策」無人驗證過（RF §4 open question）。
  這是最大風險點，不先證偽就開發 = 賭博。

## ADR-007：初版不上 neuro-symbolic
- **決策**：先做「統計概率 + 去偏誤協議」兩條有證據的腿。
- **理由**：neuro-symbolic 優勢狹窄、非全面，領域尚無像樣對照效果量（RF §3）。
- **重評時機**：兩條腿驗證有效後，再評估符號推理層是否值得加。

## ADR-008：跳過正式驗證，以興趣專案形式公開
- **決策**：依專案定位（開放式興趣、無商品化、無發論文預期），跳過 ADR-006 的正式 RCT 驗證，
  直接把工具做成靜態網站公開（GitHub Pages）。
- **理由**：正式驗證的成本（招募、IRB、預先註冊）只有在「要主張療效/要發表/要商業背書」時才必要；
  作為不宣稱預測力、誠實框定的思考輔助工具，公開使用的倫理門檻低。
- **代價（誠實揭露）**：反思層能否真正改善決策**仍未驗證**。靠誠實框定（不預測、不算命、
  個體天花板提醒）與去巴納姆護欄來守住底線，而非靠療效宣稱。
- **公開版與實驗版的差異**：公開版 = 只有 A 組完整羅盤；移除隨機分組、純巴納姆 C 組、
  錯置回饋測試、研究知情同意；資料只存使用者瀏覽器（不收集）。
- **未廢除**：ADR-001~005 的誠實護欄全部保留；`experiment/` 的研究設計保留作為文件。
- **可逆性**：若日後想做正式驗證，研究鷹架仍在 `experiment/`，可隨時重啟。

## 已知陷阱清單（隨時回看）
1. 把符號當預測引擎 → 巴納姆 + 確認偏誤。
2. 把群體概率當個體命運 → 忽略 R²<0.2 天花板。
3. 以為去偏誤一勞永逸 → 遷移弱，需持續性日誌。
4. 用「使用者覺得有用」當成功指標 → 可能只是巴納姆，必須測客觀校準度。
